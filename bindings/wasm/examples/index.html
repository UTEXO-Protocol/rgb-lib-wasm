Quick Start Guide

The typical rgb-lib workflow follows these steps. Each links to the relevant example page.

1 Generate Keys

Create a new BIP39 mnemonic and derive account xpubs for vanilla (Bitcoin) and colored (RGB) keychains.

const keys = generateKeys("regtest"); // { mnemonic, account_xpub_vanilla, account_xpub_colored, master_fingerprint }
2 Create Wallet

Instantiate a wallet from the generated keys. State is held in-memory and persisted to IndexedDB.

const walletData = { data_dir: ":memory:", bitcoin_network: "Regtest", database_type: "Sqlite", max_allocations_per_utxo: 5, account_xpub_vanilla: keys.account_xpub_vanilla, account_xpub_colored: keys.account_xpub_colored, mnemonic: keys.mnemonic, master_fingerprint: keys.master_fingerprint, vanilla_keychain: null, supported_schemas: ["Nia", "Ifa"], }; const wallet = await WasmWallet.create(JSON.stringify(walletData));
3 Go Online

Connect the wallet to an Esplora indexer. Returns an Online object used by subsequent operations.

const online = await wallet.goOnline(true, "http://127.0.0.1:8094/regtest/api");
4 Fund the Wallet

Get a Bitcoin address and send funds to it (on regtest, use the helper service).

const address = await wallet.getAddress(); // Fund via regtest helper or external wallet
5 Create UTXOs

Allocate colored UTXOs for RGB operations. Required before issuing or receiving assets.

await wallet.createUtxos(online, true, null, null, 1.5);
6 Issue an Asset

Issue an NIA (fungible) or IFA (inflatable fungible) asset on the RGB protocol.

const asset = await wallet.issueAssetNia( online, "ticker", "My Token", 0, [1000] );
7 Receive Assets

Generate a blind or witness receive to accept incoming RGB transfers.

const receive = await wallet.blindReceive( null, null, null, ["transport_endpoints"], 1 );

Run All (Automated)

Execute a full end-to-end flow automatically. "Full Flow" requires a running regtest environment; "Offline Only" tests key generation, wallet creation, and error handling without network access.

Regtest Setup Guide

    

      The full flow and most Bitcoin/RGB operations require a local regtest environment.
      Run the following from the repository root:
    

    
# Start the regtest Docker containers (bitcoind, esplora, rgb-proxy, vss)
bash tests/regtest.sh start

# Verify services are running:
#   Esplora API:    http://127.0.0.1:8094/regtest/api
#   RGB Proxy:      http://127.0.0.1:3000/json-rpc
#   Regtest Helper: http://127.0.0.1:8080
#   VSS:            http://127.0.0.1:8082/vss

# Serve the WASM examples (from the bindings/wasm directory):
python3 -m http.server 8888 --directory bindings/wasm
# Then open http://127.0.0.1:8888/examples/index.html

# When done, stop the regtest containers:
bash tests/regtest.sh stop
